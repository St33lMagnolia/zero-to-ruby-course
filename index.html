Singleton classes

(Eigenclasses/Metaclasses/Virtual Classes/Ghost classes)

  • It's an anonymous class that you can't instantiate object from it
  • It's hidden
  • It is created automatically by the interpreter when you refer to it at the first time
  • Except from those above, it works like normal classes Picture of the singleton class

self (current object)

  • Read-only variable
  • Has 2 significant roles in a running Ruby program:
    self controls how Ruby finds instance variables.
    self plays a vital role in method calling.

Understanding Ruby's method lookup

  • Find out what is self in current line of executing program
  • Ruby methods are looked up in the following order:

Ruby methods are looked up in the following order

  • Methods defined in the object's singleton class (i.e. the object itself)
  • Modules mixed into the singleton class in reverse order of inclusion
  • Methods defined by the object's class
  • Modules included into the object's class in reverse order of inclusion
  • Methods defined by the object's superclass.

Duck Typing

  • In Ruby, we rely less on the type (or class) of an object and more on its capabilities.
  • Duck Typing means an object type is defined by what it can do, not by what it is.

Basic metaprogramming and DSL

Metaprogramming

  • Wikipedia: "Metaprogramming is the writing of computer programs that write or manipulate other programs (or themselves) as their data, or that do part of the work at compile time that would otherwise be done at runtime."
  • Code that writes code.

Example

class Person
  attr_accessor :name, :age
end

Getters and Setters from Java style

class Person
  def name
    @name
  end

  def name=(value)
    @name = value
  end

  def age
    @age
  end

  def age=(value)
    @age = value
  end
end

Then change it into

class Person
  %w(name age).each do |method_name|
    define_method(method_name) do
      instance_variable_get("@#{method_name}")
    end

    define_method("#{method_name}=") do |value|
      instance_variable_set("@#{method_name}", value)
    end
  end
end

Finally

class Person
  attr_accessor :name, :age
end

Domain Specific Language(DSL)

Martin Fowler: The basic idea of a domain specific language (DSL) is a computer language that's targeted to a particular kind of problem, rather than a general purpose language that's aimed at any kind of software problem. Domain specific languages have been talked about, and used for almost as long as computing has been done.

Two kinds of DSL

Internal DSL

use a host language to give the host language the feel of a particular language.

External DSL

have their own custom syntax and you write a full parser to process them.
